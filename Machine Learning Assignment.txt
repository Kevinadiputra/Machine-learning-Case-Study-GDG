Case Study “An Actually Working Segmentation”

Tujuan: Membuat model Computer Vision yang secara spesifik dapat melakukan segmentasi (membuat outline/masking) pada dataset (dengan minimum 4 kelas) dalam gambar, kemudian men-deploy model tersebut sebagai aplikasi web yang dapat diakses publik.
Dataset:
? Anda wajib mencari Segmentation Dataset (gambar + label polygon/masks) dengan tipe dataset apapun.
? Syarat: Anda harus menggunakan minimal 4 kelas yang berbeda.
? Rekomendasi Sumber: Roboflow Universe, Kaggle (cari dataset Segmentation), atau lakukan anotasi manual menggunakan CVAT/LabelImg.
Instruksi:
1. Landasan Teori (Persona Core Team):
? Bayangkan Anda adalah anggota Core Team yang sedang mengajar anggota baru. Sertakan bagian Markdown di dalam notebook Anda yang menjelaskan:
? Perbedaan teknis antara Object Detection (Bounding Box) dan Instance Segmentation (Pixel Mask).
? Bagaimana komputer memproses gambar untuk memprediksi "mask" (misalnya: bagaimana model membedakan batas objek dengan latar belakang).
? Konsep Transfer Learning dalam Segmentasi (contoh: menggunakan bobot pre-trained dari dataset COCO pada arsitektur YOLOv8-seg atau Mask R-CNN).
2. Implementasi Proyek (Jupyter Notebook):
? Buat notebook (misal: Training_Segmentation.ipynb) yang mencakup proses end-to-end:
? Data Preprocessing: Tunjukkan proses resize gambar dan penanganan augmentasi mask.
? Training & Evaluasi:
? Lakukan training pada model.
? Syarat Metrik: Jangan gunakan "Akurasi" biasa. Anda wajib mengevaluasi menggunakan mAP (mean Average Precision) atau IoU (Intersection over Union).
? Target: Model harus mencapai mAP@0.5 > 75% atau menghasilkan mask yang akurat secara visual pada data tes.
? Pemilihan Model: Analisis hasilnya dan pilih satu model terbaik untuk deployment. Jelaskan alasan Anda (trade-off antara kecepatan vs presisi).
3. Deployment (Streamlit):
? Buat file Python bernama app.py.
? Aplikasi harus mengizinkan user untuk mengupload gambar (JPG/PNG).
? Aplikasi harus memuat model terbaik yang sudah Anda latih.
? Output: Tampilkan gambar asli bersandingan dengan hasil segmentasi (menampilkan overlay mask/polygon pada bola yang terdeteksi).
4. Dokumentasi (GitHub & DevOps):
? Upload proyek Anda ke repository GitHub publik.
? README.md: Wajib ada. File ini harus memuat:
? Judul & Deskripsi Proyek.
? Instruksi "Cara Menjalankan" (misal: pip install -r requirements.txt, streamlit run app.py).
? Penjelasan singkat arsitektur model yang digunakan.
? Screenshot aplikasi Streamlit yang sedang berjalan.
? requirements.txt: Daftar semua library Python yang dibutuhkan.
? .gitignore: Pastikan Anda tidak mengupload file sistem atau dataset yang berukuran besar secara tidak perlu.
